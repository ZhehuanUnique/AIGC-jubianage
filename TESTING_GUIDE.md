## 🧪 测试步骤

### 1. 清除浏览器缓存

**重要**：确保加载最新代码

- **Windows/Linux**: 按 `Ctrl + Shift + R` 或 `Ctrl + F5`
- **Mac**: 按 `Cmd + Shift + R`

或者：
1. 打开开发者工具（F12）
2. 右键点击刷新按钮
3. 选择"清空缓存并硬性重新加载"

### 2. 测试分辨率切换功能

1. **打开网站**：访问你的网站（如 `jubianai.cn`）
2. **查找分辨率按钮**：
   - 应该在视频生成界面的控制栏
   - 位置：在"分辨率:"标签旁边
   - 应该看到两个按钮：**720P** 和 **1080P**
3. **测试切换**：
   - 点击 **720P** 按钮（应该高亮显示）
   - 点击 **1080P** 按钮（应该切换高亮）
   - 确认按钮样式正确变化

### 3. 测试视频生成功能

1. **输入提示词**：
   - 在输入框中输入视频描述
   - 例如："一个3D形象的小男孩，在公园滑滑板"

2. **上传首帧（可选）**：
   - 点击首帧上传区域
   - 选择一张图片

3. **选择分辨率**：
   - 选择 **720P** 或 **1080P**
   - 确认选择正确

4. **选择时长**：
   - 选择 5秒 或 10秒

5. **生成视频**：
   - 点击"生成视频"按钮
   - 等待任务提交

### 4. 测试历史记录刷新

1. **检查初始状态**：
   - 页面加载后应该显示历史记录
   - 如果显示"暂无历史视频"，这是正常的（如果之前没有生成过）

2. **提交任务后**：
   - 任务提交后，1秒内应该刷新历史记录
   - 应该能看到新提交的任务（状态为"等待中"或"生成中"）

3. **等待视频生成**：
   - 视频生成期间，每30秒自动刷新历史记录
   - 应该能看到状态更新（从"生成中"到"已完成"）

4. **视频完成后**：
   - 视频生成完成后，应该立即刷新历史记录
   - 应该能看到完成的视频，包括视频URL

### 5. 验证视频生成

1. **检查视频状态**：
   - 在历史记录中，应该能看到视频卡片
   - 状态应该显示为"已完成"

2. **播放视频**：
   - 鼠标悬停在视频卡片上，视频应该自动播放
   - 点击视频可以查看完整视频

3. **检查分辨率**：
   - 如果选择的是 1080P，视频应该是 1920x1080
   - 如果选择的是 720P，视频应该是 1280x720

## 🔍 调试检查清单

### 如果分辨率按钮不显示：

1. **检查浏览器控制台**（F12）：
   - 查看是否有 JavaScript 错误
   - 查看是否有 Vue 渲染错误

2. **检查网络请求**：
   - 打开 Network 标签
   - 刷新页面
   - 确认所有资源都加载成功

3. **检查代码部署**：
   - 确认 Vercel 部署已完成
   - 查看部署日志，确认没有构建错误

### 如果历史记录不刷新：

1. **检查后端 API**：
   - 打开浏览器控制台
   - 查看 Network 标签
   - 确认 `/api/v1/video/history` 请求是否成功

2. **检查后端日志**：
   - 查看 Render 或后端服务的日志
   - 确认历史记录 API 正常工作

3. **检查数据库**：
   - 确认数据库连接正常
   - 确认视频记录已保存

### 如果视频生成失败：

1. **检查后端日志**：
   - 查看 Render 日志
   - 查看是否有错误信息

2. **检查 API 响应**：
   - 打开浏览器控制台
   - 查看 Network 标签
   - 查看 `/api/v1/video/generate` 的响应

3. **检查即梦 API**：
   - 确认 API Key 配置正确
   - 确认 API 配额未耗尽

## 🐛 常见问题

### Q: 分辨率按钮点击无反应

**A**: 
1. 清除浏览器缓存
2. 检查控制台错误
3. 确认代码已部署

### Q: 历史记录一直为空

**A**:
1. 检查后端 API 是否正常
2. 检查数据库连接
3. 查看浏览器控制台的网络请求

### Q: 视频生成后不显示

**A**:
1. 等待自动刷新（最多30秒）
2. 手动刷新页面
3. 检查视频状态是否为"已完成"
