# PostgreSQL 数据库设置指南

## 为什么选择 PostgreSQL？

PostgreSQL 相比 MySQL 的优势：
- ✅ **更强大的数据类型支持**：支持 JSON/JSONB、数组、地理信息等复杂数据类型
- ✅ **更好的复杂查询性能**：在处理多表 JOIN、窗口函数等复杂查询时表现更优
- ✅ **严格遵循 ACID 原则**：提供完整的事务支持，确保数据一致性
- ✅ **更好的并发控制**：MVCC（多版本并发控制）机制更先进
- ✅ **丰富的扩展生态**：支持向量数据库扩展（pgvector），适合未来实现知识库功能

## 方案一：使用 Vercel Postgres（推荐）

### 步骤 1：在 Vercel 中创建 Postgres 数据库

1. 登录 [Vercel Dashboard](https://vercel.com/dashboard)
2. 进入你的项目 `jubianai`
3. 点击 **"Storage"** 标签页
4. 点击 **"Create Database"**
5. 选择 **"Postgres"**
6. 选择计划（Hobby 计划有免费额度）
7. 选择区域（建议选择离你最近的区域）
8. 点击 **"Create"**

### 步骤 2：获取数据库连接信息

创建完成后，Vercel 会自动：
- 创建数据库实例
- 生成连接字符串
- 自动添加到项目的环境变量中

环境变量名称：
- `POSTGRES_URL` - 完整的连接字符串

### 步骤 3：验证环境变量

1. 在 Vercel 项目设置中，进入 **"Settings"** → **"Environment Variables"**
2. 确认 `POSTGRES_URL` 已自动添加
3. 如果没有，手动添加：
   - 变量名：`POSTGRES_URL`
   - 值：从 Vercel Storage 页面复制连接字符串

### 步骤 4：重新部署

环境变量添加后，Vercel 会自动重新部署。或者手动触发：
```bash
cd jubianai
vercel --prod
```

## 方案二：使用 Supabase（免费，功能更丰富）

### 步骤 1：创建 Supabase 项目

1. 访问 [Supabase](https://supabase.com/)
2. 使用 GitHub 账号登录
3. 点击 **"New Project"**
4. 填写项目信息：
   - 项目名称：`jubianai`
   - 数据库密码：设置一个强密码（保存好）
   - 区域：选择离你最近的区域
5. 点击 **"Create new project"**

### 步骤 2：获取连接字符串

1. 在 Supabase 项目中，进入 **"Settings"** → **"Database"**
2. 找到 **"Connection string"** 部分
3. 选择 **"URI"** 格式
4. 复制连接字符串（格式：`postgresql://postgres:[YOUR-PASSWORD]@db.xxx.supabase.co:5432/postgres`）

### 步骤 3：在 Vercel 中添加环境变量

1. 在 Vercel 项目设置中，进入 **"Settings"** → **"Environment Variables"**
2. 添加环境变量：
   - 变量名：`DATABASE_URL`
   - 值：从 Supabase 复制的连接字符串
   - 环境：选择 **"Production"** 和 **"Preview"**

### 步骤 4：重新部署

```bash
cd jubianai
vercel --prod
```

## 方案三：本地开发环境

### 使用 Docker 运行 PostgreSQL

```bash
docker run --name jubianai-postgres \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=jubianai \
  -p 5432:5432 \
  -d postgres:15
```

### 设置本地环境变量

创建 `.env` 文件：
```env
DB_USER=postgres
DB_PASSWORD=postgres
DB_HOST=localhost
DB_PORT=5432
DB_NAME=jubianai
```

或者使用连接字符串：
```env
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/jubianai
```

## 数据库表结构

系统会自动创建以下表：

### `assets` 表
- `id` - 主键
- `filename` - 原始文件名
- `character_name` - 人物名称
- `view_type` - 视图类型（正视图、侧视图等）
- `file_path` - 文件路径或 URL
- `file_url` - 完整 URL（如果使用云存储）
- `upload_time` - 上传时间
- `file_size` - 文件大小（字节）
- `file_type` - 文件类型（image, video 等）
- `metadata` - JSON 格式的额外元数据

## 常见问题

### Q: 数据库连接失败怎么办？
A: 检查：
1. 环境变量是否正确设置
2. 数据库是否已创建并运行
3. 连接字符串格式是否正确
4. 网络是否允许访问数据库

### Q: 如何查看数据库中的数据？
A: 
- **Vercel Postgres**：在 Vercel Dashboard → Storage → Postgres → 点击数据库名称 → 使用 SQL Editor
- **Supabase**：在 Supabase Dashboard → Table Editor 或 SQL Editor

### Q: 数据库会自动创建表吗？
A: 是的，应用启动时会自动调用 `init_db()` 创建表结构。

### Q: 如何备份数据库？
A: 
- **Vercel Postgres**：在 Vercel Dashboard 中可以导出数据
- **Supabase**：在 Settings → Database → Backups 中可以设置自动备份

## 下一步：文件存储

数据库只存储元数据，实际文件需要存储到：
- **Vercel Blob Storage**（推荐，与 Vercel 集成好）
- **AWS S3**
- **Cloudinary**
- **Supabase Storage**（如果使用 Supabase）

文件上传流程：
1. 前端上传文件到云存储
2. 获取文件 URL
3. 将文件 URL 和元数据一起保存到数据库
