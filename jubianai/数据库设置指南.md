# PostgreSQL 数据库设置指南

## 为什么选择 PostgreSQL？

PostgreSQL 相比 MySQL 的优势：
- ✅ **更强大的数据类型支持**：支持 JSON/JSONB、数组、地理信息等复杂数据类型
- ✅ **更好的复杂查询性能**：在处理多表 JOIN、窗口函数等复杂查询时表现更优
- ✅ **严格遵循 ACID 原则**：提供完整的事务支持，确保数据一致性
- ✅ **更好的并发控制**：MVCC（多版本并发控制）机制更先进
- ✅ **丰富的扩展生态**：支持向量数据库扩展（pgvector），适合未来实现知识库功能

## 方案一：使用 Neon（推荐，专为 Vercel 优化）

Neon 是 Serverless Postgres，与 Vercel 集成非常好，有免费额度。

### 步骤 1：在 Vercel Marketplace 中创建 Neon 数据库

1. 在 Vercel Dashboard 中，进入你的项目 `jubianai`
2. 点击 **"Storage"** 标签页
3. 点击 **"Create Database"** 或 **"Add"**
4. 在 **"Marketplace Database Providers"** 部分，找到 **"Neon"**（绿色图标，显示 "Serverless Postgres"）
5. 点击 **"Neon"** 卡片
6. 点击 **"Continue"** 按钮

### 步骤 2：连接 Neon 账户

1. 如果是第一次使用，会要求授权 Vercel 访问 Neon
2. 点击 **"Authorize"** 或 **"Connect"**
3. 如果没有 Neon 账户，会引导你创建（免费）

### 步骤 3：创建数据库

1. 选择或创建 Neon 项目
2. 选择数据库区域（建议选择离你最近的区域）
3. 点击 **"Create"** 或 **"Continue"**

### 步骤 4：获取连接字符串

创建完成后，Vercel 会自动：
- 创建数据库实例
- 生成连接字符串
- **自动添加到项目的环境变量中**

环境变量名称：
- `DATABASE_URL` - 完整的 PostgreSQL 连接字符串（Neon 使用这个名称）

### 步骤 5：验证环境变量

1. 在 Vercel 项目设置中，进入 **"Settings"** → **"Environment Variables"**
2. 确认 `DATABASE_URL` 已自动添加
3. 如果没有，手动添加：
   - 变量名：`DATABASE_URL`
   - 值：从 Neon Dashboard 或 Vercel Storage 页面复制连接字符串

### 步骤 6：重新部署

环境变量添加后，Vercel 会自动重新部署。或者手动触发：
```bash
cd jubianai
vercel --prod
```

## 方案二：使用 Supabase（通过 Vercel Marketplace）

Supabase 也是很好的选择，提供完整的后端服务（数据库 + 存储 + 认证）。

### 方法 A：通过 Vercel Marketplace（推荐）

1. 在 Vercel Dashboard 中，进入你的项目 `jubianai`
2. 点击 **"Storage"** 标签页
3. 点击 **"Create Database"** 或 **"Add"**
4. 在 **"Marketplace Database Providers"** 部分，找到 **"Supabase"**（深绿色闪电图标，显示 "Postgres backend"）
5. 点击 **"Supabase"** 卡片
6. 按照提示连接 Supabase 账户并创建数据库
7. Vercel 会自动添加 `DATABASE_URL` 环境变量

### 方法 B：手动创建 Supabase 项目

1. 访问 [Supabase](https://supabase.com/)
2. 使用 GitHub 账号登录
3. 点击 **"New Project"**
4. 填写项目信息：
   - 项目名称：`jubianai`
   - 数据库密码：设置一个强密码（保存好）
   - 区域：选择离你最近的区域
5. 点击 **"Create new project"**

6. 获取连接字符串：
   - 在 Supabase 项目中，进入 **"Settings"** → **"Database"**
   - 找到 **"Connection string"** 部分
   - 选择 **"URI"** 格式
   - 复制连接字符串

7. 在 Vercel 中添加环境变量：
   - 变量名：`DATABASE_URL`
   - 值：从 Supabase 复制的连接字符串
   - 环境：选择 **"Production"** 和 **"Preview"**

8. 重新部署：
```bash
cd jubianai
vercel --prod
```

## 方案三：本地开发环境

### 使用 Docker 运行 PostgreSQL

```bash
docker run --name jubianai-postgres \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=jubianai \
  -p 5432:5432 \
  -d postgres:15
```

### 设置本地环境变量

创建 `.env` 文件：
```env
DB_USER=postgres
DB_PASSWORD=postgres
DB_HOST=localhost
DB_PORT=5432
DB_NAME=jubianai
```

或者使用连接字符串：
```env
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/jubianai
```

## 数据库表结构

系统会自动创建以下表：

### `assets` 表
- `id` - 主键
- `filename` - 原始文件名
- `character_name` - 人物名称
- `view_type` - 视图类型（正视图、侧视图等）
- `file_path` - 文件路径或 URL
- `file_url` - 完整 URL（如果使用云存储）
- `upload_time` - 上传时间
- `file_size` - 文件大小（字节）
- `file_type` - 文件类型（image, video 等）
- `metadata` - JSON 格式的额外元数据

## 常见问题

### Q: 数据库连接失败怎么办？
A: 检查：
1. 环境变量是否正确设置
2. 数据库是否已创建并运行
3. 连接字符串格式是否正确
4. 网络是否允许访问数据库

### Q: 如何查看数据库中的数据？
A: 
- **Vercel Postgres**：在 Vercel Dashboard → Storage → Postgres → 点击数据库名称 → 使用 SQL Editor
- **Supabase**：在 Supabase Dashboard → Table Editor 或 SQL Editor

### Q: 数据库会自动创建表吗？
A: 是的，应用启动时会自动调用 `init_db()` 创建表结构。

### Q: 如何备份数据库？
A: 
- **Vercel Postgres**：在 Vercel Dashboard 中可以导出数据
- **Supabase**：在 Settings → Database → Backups 中可以设置自动备份

## 下一步：文件存储

数据库只存储元数据，实际文件需要存储到：
- **Vercel Blob Storage**（推荐，与 Vercel 集成好）
- **AWS S3**
- **Cloudinary**
- **Supabase Storage**（如果使用 Supabase）

文件上传流程：
1. 前端上传文件到云存储
2. 获取文件 URL
3. 将文件 URL 和元数据一起保存到数据库
