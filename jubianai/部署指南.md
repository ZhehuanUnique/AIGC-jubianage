# jubianai 生产环境部署指南

## 部署方案

jubianai 包含两个部分：
1. **Streamlit 前端**：用户界面
2. **FastAPI 后端**：API 服务

## 方案一：Streamlit Cloud + Vercel（推荐）

### 1. 部署 Streamlit 前端到 Streamlit Cloud

Streamlit Cloud 提供免费的 Streamlit 应用托管服务。

#### 步骤：

1. **准备部署文件**
   - 确保 `jubianai/requirements.txt` 包含所有依赖
   - 确保 `jubianai/frontend/app.py` 可以独立运行

2. **推送到 GitHub**
   - 确保代码已推送到 GitHub 仓库

3. **部署到 Streamlit Cloud**
   - 访问 https://streamlit.io/cloud
   - 使用 GitHub 账号登录
   - 点击 "New app"
   - 选择你的仓库和分支
   - **Main file path**: `jubianai/frontend/app.py`
   - **App URL**: 会自动生成，例如 `https://your-app.streamlit.app`

4. **配置环境变量**
   - 在 Streamlit Cloud 设置中添加环境变量：
     - `BACKEND_URL`: 你的后端 API 地址（Vercel 部署后的地址）

### 2. 部署 FastAPI 后端到 Vercel

#### 创建 Vercel 配置文件

在 `jubianai` 目录下创建 `vercel.json`：

```json
{
  "version": 2,
  "builds": [
    {
      "src": "backend/api.py",
      "use": "@vercel/python"
    }
  ],
  "routes": [
    {
      "src": "/(.*)",
      "dest": "backend/api.py"
    }
  ]
}
```

#### 步骤：

1. **安装 Vercel CLI**（如果还没有）
   ```bash
   npm i -g vercel
   ```

2. **在 jubianai 目录下部署**
   ```bash
   cd jubianai
   vercel
   ```

3. **配置环境变量**
   - 在 Vercel 项目设置中添加环境变量：
     - `API_KEY`: 你的 API Key
     - `SEEDANCE_API_ENDPOINT`: Seedance API 地址

4. **获取部署地址**
   - Vercel 会提供一个地址，例如：`https://your-api.vercel.app`

### 3. 更新前端配置

在 Streamlit Cloud 的环境变量中设置：
- `BACKEND_URL`: `https://your-api.vercel.app`

## 方案二：Railway（一体化部署，推荐）

Railway 可以同时部署前端和后端，更简单。

### 步骤：

1. **访问 Railway**
   - https://railway.app
   - 使用 GitHub 登录

2. **创建新项目**
   - 从 GitHub 仓库导入项目

3. **配置服务**
   - **后端服务**：
     - Root Directory: `jubianai`
     - Start Command: `uvicorn backend.api:app --host 0.0.0.0 --port $PORT`
     - 环境变量：添加 `API_KEY`、`SEEDANCE_API_ENDPOINT` 等
   
   - **前端服务**：
     - Root Directory: `jubianai`
     - Start Command: `streamlit run frontend/app.py --server.port $PORT --server.address 0.0.0.0`
     - 环境变量：添加 `BACKEND_URL`（后端服务的 Railway URL）

4. **获取地址**
   - Railway 会为每个服务提供独立的 URL

## 方案三：Render（免费方案）

### 步骤：

1. **访问 Render**
   - https://render.com
   - 使用 GitHub 登录

2. **部署后端（Web Service）**
   - 新建 Web Service
   - 连接 GitHub 仓库
   - Root Directory: `jubianai`
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `uvicorn backend.api:app --host 0.0.0.0 --port $PORT`
   - 环境变量：添加所需配置

3. **部署前端（Web Service）**
   - 新建 Web Service
   - 连接 GitHub 仓库
   - Root Directory: `jubianai`
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `streamlit run frontend/app.py --server.port $PORT --server.address 0.0.0.0`
   - 环境变量：添加 `BACKEND_URL`

## 推荐配置

### 环境变量

**后端（FastAPI）**：
- `API_KEY`: 你的 API Key
- `SEEDANCE_API_ENDPOINT`: Seedance API 地址
- `HOST`: `0.0.0.0`（生产环境）
- `PORT`: 由平台自动分配

**前端（Streamlit）**：
- `BACKEND_URL`: 后端服务的生产环境地址
- `STREAMLIT_SERVER_PORT`: 由平台自动分配
- `STREAMLIT_SERVER_ADDRESS`: `0.0.0.0`

## 更新主页面链接

部署完成后，需要更新主页面的链接地址：

在 `index.html` 和 `main.js` 中，将 `http://localhost:8501` 替换为生产环境的 Streamlit 地址。

## 安全建议

1. **CORS 配置**：在生产环境中，限制 `allow_origins` 为特定域名
2. **API Key**：使用环境变量，不要硬编码
3. **HTTPS**：确保所有服务都使用 HTTPS

## 测试

部署完成后：
1. 访问 Streamlit 前端地址
2. 测试视频生成功能
3. 检查后端 API 是否正常工作
4. 测试主页面的链接是否正常
