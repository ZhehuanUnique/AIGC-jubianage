# 即梦 API 并发限制处理指南

## 🔴 错误信息

```
code: 50430
message: "Request Has Reached API Concurrent Limit, Please Try Later"
```

**含义**: 即梦 API 有并发请求限制，当前请求数已达到上限。

## ✅ 解决方案

### 方案 1: 等待后重试（推荐）

1. **等待一段时间**
   - 即梦 API 的并发限制是临时的
   - 等待其他任务完成后，限制会自动解除
   - 建议等待 **1-5 分钟** 后重试

2. **在前端重试**
   - 点击"生成视频"按钮重试
   - 或刷新页面后重新生成

### 方案 2: 检查是否有其他任务在运行

1. **查看历史记录**
   - 在前端查看是否有其他视频正在生成
   - 等待这些任务完成后再生成新视频

2. **取消其他任务**（如果可能）
   - 如果有失败的任务，可以尝试删除
   - 释放并发资源

### 方案 3: 实现请求队列（未来优化）

可以添加请求队列功能：
- 将请求放入队列
- 按顺序处理，避免并发限制
- 自动重试机制

## 📝 即梦 API 限制说明

### 并发限制
- 即梦 API 对同一账户有并发请求限制
- 限制数量取决于你的账户类型和套餐
- 超过限制会返回 50430 错误

### 建议
1. **避免同时生成多个视频**
   - 一次只生成一个视频
   - 等待完成后再生成下一个

2. **实现请求队列**（未来）
   - 将请求排队处理
   - 自动管理并发数量

3. **监控并发状态**
   - 在前端显示当前任务状态
   - 提示用户等待

## 💡 临时解决方案

如果频繁遇到并发限制：

1. **检查账户类型**
   - 确认你的即梦 API 账户类型
   - 考虑升级套餐以获得更高并发限制

2. **优化请求频率**
   - 减少同时发起的请求
   - 增加请求间隔时间

3. **使用队列系统**
   - 实现请求队列
   - 控制并发数量

## 🆘 如果问题持续

如果频繁遇到并发限制错误：

1. **检查账户状态**
   - 确认账户是否正常
   - 检查是否有其他应用在使用同一账户

2. **联系即梦 API 支持**
   - 咨询并发限制的具体数量
   - 了解是否可以提升限制

3. **实现重试机制**
   - 自动重试失败的请求
   - 使用指数退避策略
