# 视频帧率和分辨率提升说明

## 📊 当前实现

### 分辨率（Resolution）

**已实现：**
- ✅ **720P**：通过 `req_key = "jimeng_i2v_first_v30"` 或 `"i2v_first_tail_v30_jimeng"`
- ✅ **1080P**：通过 `req_key = "jimeng_i2v_first_v30_1080"` 或 `"i2v_first_tail_v30_1080_jimeng"`

**实现方式：**
- 通过接入即梦 API 的不同端点实现
- 720P 和 1080P 是不同的 API 接口（不同的 `req_key`）
- 前端可以通过切换按钮选择分辨率

**提升更高分辨率（如 4K）：**
- ❌ **需要即梦 API 提供新的接口**
- 如果即梦 API 未来提供 4K 接口，需要：
  1. 添加新的 `req_key`（如 `jimeng_i2v_first_v30_4k`）
  2. 在代码中添加对应的处理逻辑
  3. 更新前端分辨率选择器

---

### 帧率（FPS）

**当前状态：**
- ⚠️ **即梦 API 不支持自定义帧率**
- API 只支持固定的帧数：**121帧（5秒）** 或 **241帧（10秒）**
- 输出视频的帧率是固定的（通常是 24fps）

**代码实现：**
```python
# 计算 frames（总帧数）
calculated_frames = request.duration * request.fps
if calculated_frames <= 121:
    frames = 121  # 5秒
elif calculated_frames <= 241:
    frames = 241  # 10秒
```

**注意：**
- `request.fps` 参数虽然存在，但**不会影响最终输出帧率**
- 即梦 API 的输出帧率是固定的，无法通过 API 参数改变

## 🚀 如何提升帧率和分辨率

### 方法 1: 等待即梦 API 更新（推荐）

**分辨率：**
- 如果即梦 API 未来提供更高分辨率接口（如 4K），直接接入新接口即可
- 需要查看即梦 API 官方文档，确认是否有新接口

**帧率：**
- 如果即梦 API 未来支持自定义帧率，更新 API 调用参数即可
- 目前即梦 API 不支持自定义帧率

---

### 方法 2: 后处理技术（帧率提升）

**视频插帧技术（Frame Interpolation）：**

如果即梦 API 输出的是 24fps，想要提升到 60fps 或更高，可以使用视频插帧技术：

1. **使用 AI 视频插帧工具：**
   - RIFE（Real-Time Intermediate Flow Estimation）
   - DAIN（Depth-Aware Video Frame Interpolation）
   - FILM（Frame Interpolation for Large Motion）

2. **实现方式：**
   ```python
   # 示例：使用 RIFE 进行视频插帧
   # 需要安装相关库和模型
   from rife import interpolate_video
   
   # 将 24fps 视频插帧到 60fps
   output_video = interpolate_video(
       input_video="input_24fps.mp4",
       target_fps=60
   )
   ```

3. **注意事项：**
   - 需要额外的计算资源
   - 处理时间较长
   - 可能影响视频质量（取决于插帧算法）

4. **集成建议：**
   - 可以作为可选功能（用户选择是否启用）
   - 在视频生成完成后，后台处理
   - 需要额外的服务器资源

---

### 方法 3: 后处理技术（分辨率提升）

**AI 超分辨率技术（Super Resolution）：**

如果即梦 API 输出的是 1080P，想要提升到 4K，可以使用超分辨率技术：

1. **使用 AI 超分辨率工具：**
   - Real-ESRGAN
   - ESRGAN
   - Waifu2x

2. **实现方式：**
   ```python
   # 示例：使用 Real-ESRGAN 进行超分辨率
   from realesrgan import enhance_video
   
   # 将 1080P 视频提升到 4K
   output_video = enhance_video(
       input_video="input_1080p.mp4",
       scale=2  # 2倍放大（1080P -> 4K）
   )
   ```

3. **注意事项：**
   - 需要额外的计算资源
   - 处理时间较长（视频比图片慢很多）
   - 可能影响视频质量（取决于算法）

4. **集成建议：**
   - 可以作为可选功能（"超清"按钮）
   - 在视频生成完成后，后台处理
   - 需要额外的服务器资源

## ❓ 常见问题

**Q: 可以生成 60fps 的视频吗？**
A: 目前不行。即梦 API 输出帧率是固定的（通常是 24fps）。如果需要 60fps，需要使用视频插帧技术进行后处理。

**Q: 可以生成 4K 视频吗？**
A: 目前不行。即梦 API 最高支持 1080P。如果需要 4K，需要使用超分辨率技术进行后处理，或等待即梦 API 提供 4K 接口。

**Q: 后处理会影响视频质量吗？**
A: 取决于使用的算法。AI 插帧和超分辨率通常能保持较好的质量，但可能不如原生生成的效果。

**Q: 后处理需要多长时间？**
A: 取决于视频长度和服务器性能。通常 5 秒视频的插帧或超分辨率需要 1-5 分钟。
