# 视频生成速度说明

## ⏱️ 视频生成时间

视频生成速度主要取决于 **即梦 AI API** 的处理能力，这是 API 服务端的限制，无法通过前端优化来加速。

### 典型生成时间

- **5秒视频**：通常需要 **1-3 分钟**
- **10秒视频**：通常需要 **2-5 分钟**
- **15秒视频**：通常需要 **3-7 分钟**

### 影响因素

1. **视频时长**：时长越长，生成时间越长
2. **分辨率**：1080P 比 720P 稍慢
3. **首尾帧**：有首尾帧的视频可能需要更多处理时间
4. **API 负载**：即梦 AI 服务器的当前负载情况
5. **网络延迟**：与 API 服务器的网络连接质量

## 🔄 当前优化措施

### 1. 前端轮询机制

- **轮询间隔**：每 5 秒检查一次状态
- **最大轮询次数**：60 次（约 5 分钟）
- **自动刷新历史记录**：每 30 秒刷新一次，确保状态更新

### 2. 状态提示优化

- 显示"生成中"状态
- 显示已等待时间
- 提供预计完成时间提示

### 3. 重试机制

- API 调用失败时自动重试（最多 3 次）
- 每次重试间隔递增（2秒、4秒、6秒）

## ⚠️ 无法优化的部分

以下部分受 API 限制，无法通过前端优化：

1. **API 处理速度**：即梦 AI 的视频生成速度由服务端决定
2. **队列等待时间**：如果 API 服务器繁忙，可能需要排队
3. **网络传输**：视频文件从 API 服务器传输到后端需要时间

## 💡 用户体验优化建议

### 已实现

- ✅ 显示详细的生成状态
- ✅ 显示已等待时间
- ✅ 自动刷新历史记录
- ✅ 生成完成后自动更新

### 可考虑的未来优化

1. **进度百分比**：如果 API 提供进度信息，可以显示百分比
2. **预计完成时间**：基于历史数据估算完成时间
3. **后台生成**：允许用户关闭页面，稍后查看结果
4. **通知提醒**：生成完成后发送通知

## 🔍 如何判断生成是否成功

### 成功标志

- 状态从 "生成中" 变为 "已完成"
- 视频卡片显示视频预览
- 可以播放视频

### 失败标志

- 状态显示 "生成失败"
- 错误信息提示
- 超过 5 分钟仍未完成（可能超时）

## 📊 监控和调试

### 查看生成状态

1. **浏览器控制台**：查看日志信息
2. **历史记录**：查看视频状态
3. **后端日志**：查看 API 调用情况

### 常见问题

**Q: 为什么生成这么慢？**
A: 这是即梦 AI API 的正常处理时间，视频生成需要大量计算资源。

**Q: 可以加速吗？**
A: 无法通过前端优化加速，这是 API 服务端的限制。

**Q: 生成失败怎么办？**
A: 可以重新生成，或检查网络连接和 API 配置。

## 🎯 总结

视频生成速度主要由 **即梦 AI API** 决定，前端已实现：
- ✅ 实时状态更新
