# 生产环境配置说明

## 部署 jubianai 后的配置步骤

### 1. 部署 jubianai 应用

选择以下任一方案部署：

#### 方案 A：Streamlit Cloud + Vercel（推荐）
- Streamlit 前端 → Streamlit Cloud
- FastAPI 后端 → Vercel Serverless Functions

#### 方案 B：Railway（一体化）
- 同时部署前端和后端到 Railway

#### 方案 C：Render（免费）
- 分别部署前端和后端到 Render

详细步骤见：`jubianai/部署指南.md`

### 2. 获取部署地址

部署完成后，你会得到：
- **前端地址**：例如 `https://your-app.streamlit.app`
- **后端地址**：例如 `https://your-api.vercel.app`

### 3. 配置主页面链接

有两种方式配置主页面链接：

#### 方式一：通过环境变量（推荐）

在 Vercel 项目设置中添加环境变量：
- 变量名：`JUBIANAI_AGENT_URL`
- 变量值：你的 Streamlit 前端地址（例如：`https://your-app.streamlit.app`）

然后在 `index.html` 的 `<head>` 部分添加：

```html
<script>
  // 从环境变量获取地址（Vercel 会自动注入）
  window.JUBIANAI_AGENT_URL = 'https://your-app.streamlit.app';
</script>
```

#### 方式二：直接修改代码

在 `main.js` 中，找到 `getAgentUrl` 函数，修改生产环境地址：

```javascript
// 检查是否在生产环境（非 localhost）
if (window.location.hostname !== 'localhost' && window.location.hostname !== '127.0.0.1') {
  // 替换为你的实际 Streamlit 地址
  return 'https://your-app.streamlit.app';
}
```

### 4. 配置环境变量

#### Vercel 环境变量（主页面项目）

在 Vercel 项目设置中添加：
- `JUBIANAI_AGENT_URL`: Streamlit 前端地址

#### Streamlit Cloud 环境变量

在 Streamlit Cloud 设置中添加：
- `BACKEND_URL`: FastAPI 后端地址
- `ENV`: `production`

#### FastAPI 后端环境变量（Vercel/Railway/Render）

- `API_KEY`: 你的 API Key
- `SEEDANCE_API_ENDPOINT`: Seedance API 地址
- `ALLOWED_ORIGINS`: 允许的域名（例如：`https://jubianai.cn,https://your-app.streamlit.app`）
- `ENV`: `production`

### 5. 测试

1. 访问主页面：`https://jubianai.cn`
2. 鼠标移动到"剧变时代"
3. 点击悬浮窗口或标题
4. 应该在新标签页打开 Streamlit 应用

### 6. 安全配置

#### CORS 配置

在生产环境中，后端 API 的 CORS 应该限制为特定域名：

```python
ALLOWED_ORIGINS = [
    "https://jubianai.cn",
    "https://www.jubianai.cn",
    "https://your-app.streamlit.app"
]
```

#### HTTPS

确保所有服务都使用 HTTPS。

## 快速配置脚本

部署完成后，运行以下命令更新配置：

```bash
# 1. 更新主页面中的地址
# 编辑 main.js，替换生产环境地址

# 2. 提交更改
git add .
git commit -m "配置生产环境地址"
git push
```

## 故障排查

### 问题：点击后无法打开页面

1. 检查 `JUBIANAI_AGENT_URL` 环境变量是否正确设置
2. 检查 Streamlit 应用是否正常运行
3. 检查浏览器控制台是否有错误

### 问题：前端无法连接后端

1. 检查 `BACKEND_URL` 环境变量是否正确
2. 检查后端 API 是否正常运行
3. 检查 CORS 配置是否允许前端域名

### 问题：CORS 错误

1. 检查后端的 `ALLOWED_ORIGINS` 配置
2. 确保前端地址在允许列表中
3. 检查环境变量是否正确设置
