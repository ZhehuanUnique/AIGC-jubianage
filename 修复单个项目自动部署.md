# 修复单个项目的自动部署问题

## 问题分析

- ✅ 其他项目自动部署正常
- ❌ 只有 `AIGC-jubianage` 项目自动部署不工作
- 说明：不是 Vercel 账号或 GitHub 连接的问题，而是这个特定项目的配置问题

## 可能的原因

1. **这个项目的 GitHub Webhook 失效**
   - 可能被删除或失效
   - 需要重新创建

2. **项目设置中的自动部署被关闭**
   - 需要检查项目特定的设置

3. **项目的 Git 集成有问题**
   - 需要重新连接这个项目的 GitHub

## 修复步骤

### 步骤一：重新连接这个项目的 GitHub

1. **在 Vercel 中**
   - 进入 `AIGC-jubianage` 项目（不是其他项目）
   - 点击 **Settings** → **Git**

2. **断开并重新连接**
   - 点击 **"Disconnect"** 按钮
   - 等待几秒，确认已断开
   - 点击 **"Connect Git Repository"**
   - 选择 **GitHub**
   - 找到并选择 `ZhehuanUnique/AIGC-jubianage`
   - 确认连接

3. **这会重新创建这个项目的 Webhook**
   - Vercel 会为这个项目重新创建 GitHub Webhook
   - 自动部署应该会恢复

### 步骤二：检查项目特定的设置

1. **在 Vercel 中**
   - 进入 `AIGC-jubianage` 项目
   - 点击 **Settings** → **General**

2. **检查设置**
   - **Production Branch**: 应该是 `main`
   - **Auto-deploy**: 应该已启用
   - 如果被关闭，启用它

### 步骤三：检查 GitHub Webhook

1. **在 GitHub 中**
   - 访问：https://github.com/ZhehuanUnique/AIGC-jubianage/settings/hooks
   - 查看是否有 Vercel 的 Webhook
   - 检查 Webhook 的状态

2. **如果 Webhook 不存在或失效**
   - 在 Vercel 中重新连接 GitHub（步骤一）
   - 这会自动创建新的 Webhook

### 步骤四：验证自动部署

重新连接后：

1. **创建一个测试提交**
   ```bash
   echo "test" >> test-vercel-auto.txt
   git add test-vercel-auto.txt
   git commit -m "Test: Verify auto-deploy for this project"
   git push origin main
   ```

2. **观察 Vercel**
   - 1-2 分钟内应该自动创建新部署
   - 部署应该显示 "main [commit-hash] Test: Verify..."
   - 而不是 "Redeploy of..."

## 推荐操作

**立即执行**：

1. **在 Vercel 中重新连接这个项目的 GitHub**
   - 进入 `AIGC-jubianage` 项目
   - Settings → Git → Disconnect
   - 然后 Connect Git Repository
   - 重新选择仓库

2. **检查 General 设置**
   - Settings → General
   - 确认 Auto-deploy 已启用

3. **创建测试提交验证**
   - 推送一个小的更改
   - 观察是否自动部署

## 为什么只有这个项目有问题？

可能的原因：
- 这个项目的 Webhook 被意外删除
- 项目设置被更改
- 之前的部署错误导致 Webhook 失效

重新连接 GitHub 应该能解决这个问题。
