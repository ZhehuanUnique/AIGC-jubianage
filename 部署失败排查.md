# 部署失败排查指南

## 当前状态

- ✅ 代码语法检查通过
- ❌ Vercel 部署失败（从截图看）

## 排查步骤

### 步骤一：查看 Vercel 部署日志

1. **在 Vercel 中**
   - 访问 https://vercel.com
   - 进入 `AIGC-jubianage` 项目
   - 点击失败的部署（显示 "Deployment failed"）
   - 查看 **"Build Logs"** 或 **"Function Logs"**

2. **查找错误信息**
   - 查看具体的错误消息
   - 可能是：
     - 语法错误
     - 构建错误
     - 配置错误
     - 依赖问题

### 步骤二：检查常见问题

#### 1. 检查 `vercel.json` 配置

确保 `vercel.json` 格式正确：
- JSON 格式有效
- 没有不支持的属性
- 路径配置正确

#### 2. 检查文件大小

Vercel 对文件大小有限制：
- 单个文件最大 50MB
- 总部署大小有限制

检查是否有大文件：
```bash
# 检查大文件
find . -type f -size +10M
```

#### 3. 检查构建命令

对于静态网站，Vercel 应该自动检测，不需要构建命令。

### 步骤三：尝试修复

根据错误信息，可能需要：

1. **修复语法错误**
   - 如果日志显示语法错误，修复它

2. **修复配置错误**
   - 如果日志显示配置错误，修复 `vercel.json`

3. **清理缓存**
   - 在 Vercel 中：Settings → General → Clear Build Cache

### 步骤四：手动触发部署

如果自动部署不工作：

1. **在 Vercel 中手动触发**
   - Deployments → 点击 "..." → Redeploy

2. **或者使用 CLI**
   ```bash
   vercel --prod
   ```

## 需要的信息

请提供：
1. **Vercel 部署日志中的具体错误信息**
2. **错误发生在哪个阶段**（Building / Deploying）
3. **是否有任何警告信息**

这样我可以更准确地帮你修复问题。
