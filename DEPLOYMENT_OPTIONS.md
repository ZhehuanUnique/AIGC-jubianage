# 部署平台选择指南

## Render 免费计划说明

### 实际限制
- ✅ **可以创建多个服务**（没有数量限制）
- ⚠️ **每个服务在15分钟无活动后休眠**
- ⚠️ **每月750小时总运行时间**（所有服务共享）
- ⚠️ **Blueprint 只是配置方式，不是限制**

### 解决方案

#### 方案 1：继续使用 Render，手动创建服务（推荐）

不使用 Blueprint，直接创建多个 Web Service：

1. 每个项目单独创建 Web Service
2. 手动配置构建和启动命令
3. 所有服务共享750小时/月

**优点**：免费，可以多个项目
**缺点**：服务会休眠，首次访问需要几秒唤醒

#### 方案 2：使用 Railway（推荐用于多项目）

**免费计划特点**：
- ✅ 每月 $5 免费额度
- ✅ 可以创建多个项目
- ✅ 服务不会休眠（有额度时）
- ✅ 自动部署，配置简单

**配置方式**：
- 使用 `railway.json` 或直接在 Dashboard 配置
- 支持 Python、Node.js 等多种环境

#### 方案 3：使用 Fly.io

**免费计划特点**：
- ✅ 每月 3 个共享 CPU 实例
- ✅ 160GB 出站流量/月
- ✅ 服务不会休眠
- ✅ 全球边缘部署

**配置方式**：
- 使用 `fly.toml` 配置文件
- 支持多种运行时

#### 方案 4：使用 Vercel（如果解决静态文件问题）

**免费计划特点**：
- ✅ 无限项目
- ✅ 自动部署
- ✅ 全球 CDN
- ✅ 服务不会休眠

**之前的问题**：静态文件部署问题
**解决方案**：可以尝试将 API 移到标准位置，或使用不同的项目结构

## 针对你的情况

### 如果只有 1-2 个项目
**推荐：继续使用 Render**
- 手动创建 Web Service（不使用 Blueprint）
- 配置简单，免费额度够用

### 如果有 3+ 个项目
**推荐：Railway**
- 免费额度更灵活
- 服务不休眠
- 配置简单

### 如果需要全球加速
**推荐：Fly.io 或 Vercel**
- 边缘部署，访问速度快
