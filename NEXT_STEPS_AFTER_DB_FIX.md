## 🧪 测试步骤

### 1. 等待部署完成

代码已提交，等待 Render 重新部署（通常 1-2 分钟）

### 2. 生成一个新视频

1. 打开网站
2. 输入视频描述
3. 选择分辨率（720P 或 1080P）
4. 选择时长
5. 点击"生成视频"

### 3. 检查 Render 日志

在 Render Dashboard 的服务日志中，查找：

**如果保存成功，应该看到：**
```
🔍 准备保存视频生成记录: task_id=xxx, user_id=1
🔍 创建 VideoGeneration 对象: task_id=xxx, user_id=1
🔍 添加到数据库会话...
🔍 提交事务...
🔍 刷新对象...
✅ 记录创建成功: id=xxx
✅ 视频生成记录已保存: task_id=xxx, user_id=1, record_id=xxx
```

**如果保存失败，应该看到：**
```
❌ 创建记录时出错: [具体错误信息]
❌ 保存视频生成记录失败: [错误信息]
```

### 4. 检查 Supabase 表

1. 在 Supabase Table Editor 中
2. 打开 `video_generations` 表
3. 应该能看到新记录

### 5. 检查前端历史记录

1. 刷新网站页面
2. 应该能看到新生成的视频记录

## 🔍 如果仍然失败

如果保存仍然失败，请查看 Render 日志中的错误信息，告诉我：

1. **错误类型**：是什么错误？
2. **错误信息**：完整的错误信息是什么？
3. **错误位置**：在哪个步骤失败？

常见错误：
- **外键约束错误**：`user_id` 必须引用 `users.id`
- **唯一约束错误**：`task_id` 必须唯一
- **数据类型错误**：字段类型不匹配
- **NOT NULL 约束错误**：必填字段为空
