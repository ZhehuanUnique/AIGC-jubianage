# 历史记录问题调试指南

## 🔍 问题诊断

如果看不到历史记录，可能的原因：

### 1. 数据库未配置

**检查方法**：
1. 查看后端日志（Render Dashboard）
2. 查找以下消息：
   - `⚠️ 数据库未配置，跳过保存历史记录（SUPABASE_DB_URL 未设置）`
   - `数据库未配置，返回空历史记录`

**解决方案**：
- 配置 `SUPABASE_DB_URL` 环境变量
- 参考 `jubianai/RENDER_DB_SETUP.md`

### 2. 数据库保存失败

**检查方法**：
1. 查看后端日志
2. 查找错误信息：
   - `❌ 保存视频生成记录失败`
   - 数据库连接错误

**解决方案**：
- 检查数据库连接字符串是否正确
- 检查数据库表是否已创建
- 参考 `jubianai/supabase_init.sql`

### 3. 前端请求失败

**检查方法**：
1. 打开浏览器控制台（F12）
2. 查看 Network 标签
3. 查找 `/api/v1/video/history` 请求
4. 查看请求状态和响应

**可能的问题**：
- 404: API 路径错误
- 500: 后端错误
- CORS 错误: 跨域问题
- 连接失败: 后端未启动

### 4. 前端筛选问题

**检查方法**：
1. 打开浏览器控制台
2. 查看日志：
   - `历史记录API响应:`
   - `历史记录加载成功:`

**可能的问题**：
- 筛选条件太严格，过滤掉了所有记录
- 时间范围筛选问题

## 🛠️ 调试步骤

### 步骤 1: 检查后端日志

1. 访问 Render Dashboard
2. 查看服务日志
3. 查找以下关键信息：
   - `✅ 视频生成记录已保存`
   - `⚠️ 数据库未配置`
   - `❌ 保存视频生成记录失败`

### 步骤 2: 检查浏览器控制台

1. 打开浏览器控制台（F12）
2. 查看 Console 标签
3. 查找以下日志：
   - `开始加载历史记录，后端URL:`
   - `历史记录API响应:`
   - `历史记录加载成功:`
   - `加载历史记录失败:`

### 步骤 3: 检查网络请求

1. 打开浏览器控制台（F12）
2. 切换到 Network 标签
3. 刷新页面
4. 查找 `/api/v1/video/history` 请求
5. 检查：
   - 请求状态码（应该是 200）
   - 响应内容（应该包含 `total` 和 `items`）

### 步骤 4: 手动测试 API

使用 curl 或 Postman 测试：

```bash
curl -X GET "https://your-backend-url/api/v1/video/history?limit=20&offset=0"
```

预期响应：
```json
{
  "total": 1,
  "items": [
    {
      "id": 1,
      "task_id": "4862050860119547250",
      "prompt": "...",
      "status": "processing",
      ...
    }
  ],
  "limit": 20,
  "offset": 0
}
```

## 🔧 快速修复

### 如果数据库未配置

1. **设置环境变量**：
   - 在 Render Dashboard 中添加 `SUPABASE_DB_URL`
   - 格式：`postgresql://user:password@host:port/database`

2. **初始化数据库**：
   - 运行 `jubianai/supabase_init.sql` 创建表

3. **重启服务**：
   - 在 Render Dashboard 中手动重启服务

### 如果前端请求失败

1. **检查后端 URL**：
   - 确认 `config.public.backendUrl` 正确
   - 检查 CORS 配置

2. **检查网络**：
   - 确认后端服务正在运行
   - 测试后端健康检查端点

## 📊 预期行为

### 正常流程：

1. ✅ 用户提交视频生成任务
2. ✅ 后端保存记录到数据库（如果配置了）
3. ✅ 前端立即刷新历史记录（1秒后）
4. ✅ 历史记录显示新任务（状态：pending/processing）
5. ✅ 每30秒自动刷新历史记录
6. ✅ 视频完成后，立即刷新并显示完成的视频

### 如果数据库未配置：

1. ⚠️ 任务提交成功
2. ⚠️ 后端跳过保存记录
3. ⚠️ 前端刷新历史记录，返回空列表
4. ⚠️ 显示"暂无历史视频"

## 💡 临时解决方案

如果暂时无法配置数据库，可以：

1. **使用内存存储**（仅用于测试）：
   - 修改后端，使用内存列表存储
   - 重启后数据会丢失

2. **使用文件存储**（简单方案）：
   - 将记录保存到 JSON 文件
   - 需要持久化存储

3. **配置 Supabase**（推荐）：
   - 免费计划足够使用
   - 参考 `jubianai/RENDER_DB_SETUP.md`
